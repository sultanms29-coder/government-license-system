منصة العلاقات العامة والحكومية — الإصدار 22 (مرحلة الحماية الأولى)

تم تطبيق تحسينات لا تكسر النظام الحالي:
1) إجبار HTTPS عند الاستضافة الفعلية، مع السماح بالتجربة المحلية.
2) منع تضمين المنصة داخل iframe للحد من Clickjacking.
3) قفل الجلسة بعد 30 دقيقة من عدم النشاط ومسح بيانات الدخول المحفوظة.
4) منع تخزين مفتاح Supabase Service Role داخل المتصفح، ومنع روابط HTTP غير المشفرة.
5) تقليل تسريب عناوين الصفحات والبيانات عبر Referrer Policy.
6) تعطيل صلاحيات المتصفح غير اللازمة مثل الكاميرا والميكروفون والموقع والدفع وUSB.
7) ملفات حماية جاهزة لـ Apache (.htaccess) وNginx وIIS (web.config).
8) منع استعراض المجلدات والوصول إلى الملفات الحساسة والنسخ الاحتياطية.
9) إضافة security.txt وقالب أولي لسياسات Supabase RLS.
10) تحديث Service Worker لمنع بقاء نسخة قديمة من التطبيق.

تنبيه صريح:
هذه مرحلة حماية للنسخة الحالية الثابتة، ولا تحولها وحدها إلى نظام مؤسسي مكتمل الأمان.
الحماية الأعلى تتطلب المرحلة الثانية: Supabase Auth، جداول منفصلة بدل app_state، RLS فعلية، Backend، MFA، Audit Logs، وتخزين خاص للمرفقات.

قبل النشر:
- غيّر بيانات security.txt إلى بريد ونطاق الشركة الحقيقيين.
- طبّق ملف الإعداد الموافق للسيرفر فقط.
- لا تضع Service Role Key في أي ملف HTML أو JavaScript.
- نفّذ اختبار اختراق مصرح به على بيئة تجريبية قبل الإنتاج.
